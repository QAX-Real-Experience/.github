# ⚡ Welcome to **QAX Real Experience** by [QAXpert](https://qaxpert.com)

🚀 Un programa donde los aprendices viven la experiencia real de trabajar como **QA Automation** dentro de un equipo ágil que construye un producto real.  
Aquí la práctica ocurre en contexto: **features reales**, **sprints**, **automatización**, **CI/CD**, **pair review** y trabajo colaborativo con enfoque en calidad.

---
## 💡 ¿Qué es QAX Real Experience?
**QAX Real Experience** es una metodología de aprendizaje práctico basada en la vivencia real del rol de QA Automation dentro de un sprint evolutivo.

Los participantes aprenden a asegurar la calidad de una feature real aplicando:

- 🔌 **Pruebas automatizadas** sobre APIs, Web, Mobile o GraphQL  
- ⚙️ **Integración continua (CI/CD)**  
- 🐞 **Reporte y seguimiento de bugs**  
- 🔁 **Pruebas de regresión**  
- 🤝 **Pair review entre compañeros**  
- 🧩 **Trabajo en equipo orientado a calidad**  

---

## 🧭 Principios clave
| Principio | Descripción |
|-----------|-------------|
| 🎯 **Sin lecciones, solo evaluación y orientación** | Aquí no se enseña desde cero: los participantes ya deben estar listos para asumir una feature completa. |
| 🧪 **Framework libre** | Cada aprendiz trabaja con el framework que ya desarrolló en sus mentorías 1:1. |
| 🤝 **Pair Review cruzado** | Los compañeros ejecutan y validan las automatizaciones del otro, priorizando el enfoque de prueba sobre el lenguaje. |
| 🗂️ **Repositorio QAX** | Todo el trabajo vive en el ecosistema real de QAX en GitHub, con atribución para los participantes. |
| 🚀 **Experiencia real** | Se aprende qué automatizar, qué no y cómo aportar calidad sin frenar al equipo durante el sprint. |


---

## 🔄 Cómo funciona la experiencia
El programa replica un entorno **Scrum real adaptado al aprendizaje**, con ceremonias y entregas dentro de un sprint.

### Ceremonias principales
- 🗓️ **Planning / Kickoff**
- 📍 **Daily automatizada por bot**
- 🧠 **Refinamiento durante el sprint**
- ✅ **Review**
- 🔁 **Retrospectiva**

Durante cada sprint, los participantes:
- toman una feature real
- planifican la estrategia de calidad
- automatizan pruebas
- ejecutan validaciones en CI/CD
- revisan código y resultados
- participan en feedback cruzado
- presentan avances en Review
